The Green Light That Lied

Jun 2026 3 min read

CyberPanel's SSL dashboard recently told me a certificate was active. The "days remaining" column, on the same row, said -2.

Both of these things were displayed at the same time, on the same screen, and the panel was not embarrassed about it.

What "Active" actually means

I'd assumed "Active" meant the certificate was valid. It turns out it means: a file exists in the expected directory.

That's the whole check. The file could be expired, self-signed, or years old. As long as something is sitting at the path the panel expects, the status reads green. A missing indicator makes you go look. A misleading one stops you from looking.

Why the certificate expired

Let's Encrypt certificates last 90 days and renew automatically. CyberPanel uses acme.sh to handle this — except that at some point, the default certificate authority was changed from Let's Encrypt to ZeroSSL.

ZeroSSL is fine. It just requires EAB credentials before it'll accept ACME requests. CyberPanel's installer doesn't always set these up. The first manual issuance works, because the user can authenticate interactively. The nightly cron renewal fails, because it can't. Every night. For months. The dashboard reports green throughout, because the original file still exists, expired or not.

The fix

Two commands over SSH. Switch acme.sh back to Let's Encrypt as the default, then force a new issuance: 

/root/.acme.sh/acme.sh --set-default-ca --server letsencrypt

/root/.acme.sh/acme.sh --issue \
  -d example.com -d www.example.com \
  --cert-file /etc/letsencrypt/live/example.com/cert.pem \
  --key-file /etc/letsencrypt/live/example.com/privkey.pem \
  --fullchain-file /etc/letsencrypt/live/example.com/fullchain.pem \
  --webroot /home/example.com/public_html --force

Then systemctl restart lsws.

One more thing if it's a WordPress site: WordPress's rewrite rules intercept the ACME challenge URL and break validation. Add this line to the top of .htaccess: 

RewriteRule ^\.well-known/acme-challenge/ - [L]

Cost: zero. Future grief avoided: substantial.

The pattern worth noticing

Self-hosted infrastructure is usually a stack of independently maintained projects — control panel, web server, ACME client, CMS — each sensibly designed for its own users. The behaviour at the seams between them is nobody's responsibility.

This is where reliability quietly erodes. Not in any one component, but in the gaps. The pieces remain individually healthy. The composite degrades in ways that aren't anyone's bug.

A green light that confidently reports the wrong check is worse than the underlying failure. The failure is two commands to fix. The misleading indicator is what kept me from noticing for three months.

The dashboard still says Active.

It will keep saying that no matter what's true.
